## Теоретическая часть: методы оптимизации

### Обратное распространение ошибки (Backpropagation)

Backpropagation — это алгоритм вычисления градиентов функции потерь по весам сети. Сам по себе он **не является методом обучения** — он лишь эффективно считает производные с помощью цепного правила (chain rule), двигаясь от выходного слоя к входному. Далее эти градиенты передаются **оптимизатору**, который решает, как именно обновить веса.

### 1. SGD (Stochastic Gradient Descent) — базовый градиентный спуск

Простейшее правило обновления весов:

```
w = w - lr * grad
```

Где `lr` — learning rate (шаг обучения), `grad` — градиент функции потерь по весу.

**Проблемы:**
- Фиксированный шаг: слишком большой — расходится, слишком маленький — застревает
- Осцилляции в «оврагах» функции потерь (когда кривизна сильно различается по разным направлениям)
- Застревание в локальных минимумах и седловых точках

### 2. SGD с моментом (Momentum)

Добавляет «инерцию» к обновлению — учитывает не только текущий градиент, но и историю:

```
v = momentum * v_prev + lr * grad
w = w - v
```

**Идея:** мяч, катящийся с горы, набирает скорость на длинных спусках и проскакивает мелкие ямы. Параметр `momentum` (обычно 0.9) определяет, насколько сильно учитывается предыдущее направление.

**Что это даёт:**
- Ускорение сходимости на пологих участках (накопленная скорость)
- Сглаживание осцилляций в оврагах
- Способность проскочить мелкие локальные минимумы

### 3. traingdx — SGD + момент + адаптивный learning rate

Это метод из MATLAB Neural Network Toolbox, который добавляет к SGD с моментом **адаптивное изменение шага обучения**. В нашем коде реализован как `train_gdx()`.

Правила адаптации:
- Если ошибка **уменьшилась** → увеличить lr: `lr = lr * lr_inc` (lr_inc = 1.05)
- Если ошибка **выросла более чем на 4%** → откатить веса к предыдущим, уменьшить lr: `lr = lr * lr_dec` (lr_dec = 0.7)
- Если ошибка выросла, но незначительно → lr не меняется

**Зачем:** на ранних этапах обучения, когда мы далеко от минимума, можно шагать крупнее. По мере приближения к минимуму шаг автоматически уменьшается, чтобы не «перепрыгнуть» его. Откат весов при росте ошибки защищает от расходимости.

**Это метод первого порядка** — использует только градиент (первую производную).

### 4. RProp (Resilient Propagation)

RProp — метод, который учитывает только **знак** градиента, игнорируя его величину. Каждый вес имеет свой индивидуальный шаг обновления:

```
если знак градиента не изменился: step *= 1.2  (увеличить шаг)
если знак градиента изменился:    step *= 0.5  (уменьшить шаг)
w = w - sign(grad) * step
```

**Преимущества:**
- Не зависит от масштаба градиента (только знак)
- Каждый вес адаптируется независимо
- Быстрая сходимость для задач классификации
- Нет гиперпараметра learning rate в обычном смысле

### 5. trainbfg / LBFGS — квазиньютоновский метод (второго порядка)

Методы второго порядка используют информацию о **кривизне** функции потерь (вторые производные — матрица Гессе). Полный метод Ньютона обновляет веса по формуле:

```
w = w - H⁻¹ * grad
```

Где `H⁻¹` — обратная матрица Гессе. Но вычислять и хранить полный гессиан для нейросети дорого (матрица N×N, где N — число весов).

**BFGS** (Broyden–Fletcher–Goldfarb–Shanno) — приближённо строит обратный гессиан из истории градиентов, не вычисляя вторые производные напрямую.

**L-BFGS** (Limited-memory BFGS) — хранит не всю матрицу, а только последние m векторов (обычно m=10-20), что экономит память.

**Что даёт метод второго порядка:**
- Намного быстрее сходится (меньше эпох до минимума)
- Лучше справляется с «оврагами» — учитывает кривизну, а не только направление наклона
- Line search (поиск по прямой) — подбирает оптимальный шаг в найденном направлении

**Недостатки:**
- Каждая эпоха дороже по вычислениям
- Требует closure (повторный forward pass внутри оптимизатора)
- Может переобучиться быстрее (меньше шума → меньше регуляризации)

### Сравнение методов

| Характеристика | traingdx (1-й порядок) | trainbfg/LBFGS (2-й порядок) |
|---|---|---|
| Информация | Градиент (1-я производная) | Градиент + кривизна (≈2-я производная) |
| Стоимость шага | Дешёвый | Дорогой (line search, closure) |
| Число эпох | Много (тысячи) | Мало (сотни) |
| Память | O(N) | O(N·m), m ≈ 10-20 |
| Адаптация lr | Ручная (lr_inc/lr_dec) | Автоматическая (line search) |
| Риск переобучения | Ниже | Выше |

```python
def divide_rand(x, y, train_r=0.7, val_r=0.2):
    n = len(x)
    perm = np.random.permutation(n)
    n_tr = int(n * train_r)
    n_val = int(n * (train_r + val_r))
    return (x[perm[:n_tr]], y[perm[:n_tr]],
            x[perm[n_tr:n_val]], y[perm[n_tr:n_val]],
            x[perm[n_val:]], y[perm[n_val:]])
```

### Ячейка 12 — Архитектура сети

```python
class ClassificationNet(nn.Module):
    def __init__(self, hidden_size=20):
        super().__init__()
        self.hidden = nn.Linear(2, hidden_size)
        self.output = nn.Linear(hidden_size, 3)

    def forward(self, x):
        x = torch.tanh(self.hidden(x))
        x = torch.sigmoid(self.output(x))
        return x
```

Двухслойная сеть прямого распространения (feedforward):

| Слой | Вход | Выход | Активация | Параметры |
|------|------|-------|-----------|-----------|
| hidden | 2 | 20 | tanh | 20×2 весов + 20 смещений = 60 |
| output | 20 | 3 | sigmoid | 3×20 весов + 3 смещения = 63 |
| **Итого** | | | | **123 параметра** |

**Почему tanh в скрытом слое?** Выход от -1 до +1, симметричен относительно нуля — градиенты не смещены в одну сторону, что ускоряет обучение.

**Почему sigmoid на выходе?** Выход от 0 до 1 — удобно интерпретировать как «принадлежность к классу». Порог 0.5 → классификация.

### Ячейка 14 — Функция обучения (общая)

```python
def train_model(model, optimizer, X_tr, Y_tr, X_vl, Y_vl,
                epochs=1500, max_fail=1500, goal=1e-5,
                use_lbfgs=False, verbose_step=200):
```

Универсальная функция обучения с ранней остановкой. Используется и для RProp (этап 1), и для LBFGS (этап 3).

**Цикл обучения (каждая эпоха):**
1. `optimizer.zero_grad()` — обнуляем градиенты (PyTorch накапливает их)
2. `loss = criterion(model(X_tr), Y_tr)` — forward pass + MSE
3. `loss.backward()` — backpropagation (вычисление градиентов)
4. `optimizer.step()` — обновление весов (RProp или LBFGS)
5. Оценка на валидационном множестве (без градиентов — `torch.no_grad()`)

**Ранняя остановка (early stopping):**
```python
if vl_loss < best_val:
    best_val = vl_loss
    fails = 0
    best_state = ...  # запоминаем лучшие веса
else:
    fails += 1
```
Если валидационная ошибка не улучшается `max_fail` эпох подряд — останавливаем обучение и возвращаем **лучшие** веса (не последние).

**LBFGS-режим:**
```python
if use_lbfgs:
    def closure():
        optimizer.zero_grad()
        loss = criterion(model(X_tr), Y_tr)
        loss.backward()
        return loss
    tr_loss = optimizer.step(closure).item()
```
LBFGS требует `closure` — функцию, которая делает forward+backward. Это потому что LBFGS внутри вызывает closure несколько раз за один шаг (line search — поиск оптимального размера шага вдоль найденного направления).

**Критерии остановки:**
- `goal` — целевая MSE достигнута
- `max_fail` — валидационная ошибка не улучшается слишком долго
- `epochs` — максимальное число эпох


Сеть для аппроксимации: [1] → [10, tanh] → [1, linear].

**Функция `train_gdx`** — реализация метода traingdx из MATLAB:

```python
optimizer = torch.optim.SGD(model.parameters(), lr=lr, momentum=momentum)
```

Базовый оптимизатор — SGD с моментом. Поверх него добавлена **адаптация learning rate:**

```python
if tr_loss > prev_loss * 1.04:
    # Ошибка выросла более чем на 4% — откат + уменьшение lr
    model.load_state_dict(prev_state)
    for pg in optimizer.param_groups:
        pg['lr'] = max(pg['lr'] * lr_dec, 1e-7)
elif tr_loss < prev_loss:
    # Ошибка уменьшилась — увеличиваем lr

        pg['lr'] = min(pg['lr'] * lr_inc, 1.0)
```

Три сценария на каждой эпохе:
1. **Ошибка выросла сильно (>4%):** что-то пошло не так. Откатываем веса к предыдущему состоянию (`load_state_dict(prev_state)`) и уменьшаем lr в 0.7 раза. Это защита от расходимости.
2. **Ошибка уменьшилась:** всё хорошо, можно попробовать шагать крупнее. Увеличиваем lr в 1.05 раза.
3. **Ошибка выросла, но немного (<4%):** ничего не меняем, продолжаем.

## Ключевые паттерны в коде

### Early stopping
Во всех этапах используется ранняя остановка: если ошибка на валидационном множестве перестаёт уменьшаться — обучение останавливается. При этом возвращаются веса с **лучшей** валидационной ошибкой, а не последние. Это защита от переобучения.

### Batch training
Все данные подаются в сеть целиком за одну эпоху (не mini-batch). Это возможно, потому что датасеты небольшие (сотни точек). Для больших данных используют mini-batch SGD.

### model.eval() и torch.no_grad()
```python
model.eval()
with torch.no_grad():
    out = model(X)
```
`model.eval()` переключает режим (отключает dropout, batch norm — у нас их нет, но это хорошая практика). `torch.no_grad()` отключает вычисление градиентов — экономит память и время при инференсе/валидации.

### Множественная инициализация
Обучение повторяется с разными seed, выбирается лучший результат. Это важно для небольших сетей, где начальные веса сильно влияют на итоговое качество.
